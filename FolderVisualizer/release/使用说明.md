# Folder Visualizer 使用说明

## 简介

Folder Visualizer 是一款轻量级 Windows 桌面工具，用于快速生成和导出文件夹结构的可视化树形图。程序采用原生 C++ 开发，无需安装任何运行时库，开箱即用。

## 功能特性

-  快速扫描文件夹结构（支持万级文件）
-  美观的树形可视化展示（带连接线和图标）
-  智能文件分类统计（按类型、大小）
-  导出为 TXT 文本（保留完整层级结构）
-  高分辨率导出为 PNG 图片（2x 缩放，适合打印）
-  流畅的鼠标滚轮滚动浏览
-  自定义文件类型图标（支持扩展名映射）

## 系统要求

- Windows 7 或更高版本
- 无需额外安装运行库（静态链接）

## 使用方法

### 1. 启动程序

双击 `FolderVisualizer.exe` 运行程序（程序图标已嵌入，无需额外文件）。

### 2. 选择文件夹

- 点击 **“浏览...”** 按钮
- 在弹出的对话框中选择要扫描的文件夹
- 点击 **“确定”**

### 3. 开始扫描

- 点击 **“开始扫描”** 按钮
- 程序将快速扫描所选文件夹及其子文件夹
- 扫描完成后会在窗口中显示带图标的树形结构

### 4. 浏览结果

- 使用 **鼠标滚轮** 上下滚动浏览
- 文件夹以 **粗体青色** 显示，并带有 `/` 后缀
- 文件以 **常规字体** 显示，右侧显示：
  - 文件类型分类（如 `CODE`, `IMAGE` 等）
  - 文件扩展名
  - 文件大小（自动格式化为 KB/MB/GB）
  - 自定义描述（如果配置了）

### 5. 导出结果

#### 导出为文本
- 点击 **“保存 TXT”** 按钮
- 选择保存位置
- 生成包含完整目录结构的文本文件（UTF-8 编码）

#### 导出为图片（高清）
- 点击 **“保存图片”** 按钮
- 选择保存位置
- 生成 **高分辨率 PNG 图片**（宽度 3200px，高度自适应）
- 图片质量优化，适合打印和大屏展示

## 编译说明

### 使用 VS Code 构建

1. 打开项目文件夹
2. 按 `Ctrl+Shift+B` 打开构建任务
3. 选择 **"C++ Build - MinGW"**
4. 生成的可执行文件位于 `bin/` 目录

### 使用命令行构建

```bash
cd d:\AAA\QODER\FolderVisualizer2
mkdir bin
D:\msys64\ucrt64\bin\g++.exe -std=c++17 -DUNICODE -D_UNICODE -D_WIN32_WINNT=0x0601 -O2 -mwindows FolderVisualizerGUI.cpp FileTree.cpp TreeNode.cpp IconManager.cpp FileTypeManager.cpp resource.rc -lgdiplus -lcomctl32 -lshell32 -lole32 -luuid -o bin/FolderVisualizer.exe
```

### 使用 CMake 构建

```bash
cd d:\AAA\QODER\FolderVisualizer2
mkdir build
cd build
cmake .. -G "MinGW Makefiles"
cmake --build . --config Release
```

## 技术栈

- **语言**: C++17（现代特性：智能指针、Lambda、文件系统）
- **GUI 框架**: Win32 API（原生 Windows 界面）
- **图形库**: GDI+（高质量 2D 渲染）
- **构建工具**: MinGW-w64 / CMake 3.15+
- **IDE**: VS Code
- **资源编译**: Windows Resource Compiler（嵌入图标和版本信息）

## 注意事项

1. **扫描大文件夹时可能需要一些时间**，请耐心等待
2. **默认不显示隐藏文件和系统文件**
3. **每个目录最多扫描 10000 个文件**（防止卡顿）
4. **最大递归深度为 100 层**

## 常见问题

### Q: 程序无法启动？
A: 请确保使用的是 Windows 7 或更高版本。

### Q: 扫描失败？
A: 请检查：
- 路径是否存在
- 是否有权限访问该文件夹
- 路径是否包含特殊字符

### Q: 如何编译调试版本？
A: 使用 VS Code 任务 **"C++ Build - MinGW Debug"**，或在命令行中添加 `-g` 参数。

## 配置文件说明

程序支持通过 `.vscode/settings.json` 文件自定义图标映射：

```json
{
    "folderVisualizer.iconPath": "icons/",
    "folderVisualizer.defaultFolderIcon": "folder.png",
    "folderVisualizer.defaultFileIcon": "file.png",
    "folderVisualizer.defaultcppIcon": "cpp.png",
    "folderVisualizer.defaulthIcon": "h.png",
    "folderVisualizer.defaultpyIcon": "py.png"
}
```

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `iconPath` | 图标文件夹路径 | `icons/` |
| `defaultFolderIcon` | 默认文件夹图标 | `folder.png` |
| `defaultFileIcon` | 默认文件图标 | `file.png` |
| `default{扩展名}Icon` | 指定扩展名的自定义图标 | `defaultcppIcon: "cpp.png"` |

**注意**：
- 程序主图标已嵌入到可执行文件中，无需配置
- 自定义扩展名图标格式为 `default{扩展名}Icon`，如 `defaultcppIcon`、`defaulthtmlIcon`
- 图标文件需要放在 `iconPath` 指定的目录中
- 如果未配置自定义图标，程序会使用默认文件图标
